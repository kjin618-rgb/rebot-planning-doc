01 Overview

서비스 개요

🎯 해결하는 문제

소규모 카페·베이커리 사장님은 고객 재방문 유도가 중요하지만, 데이터 분석 도구를 다룰 시간·기술이 없다. 종이 스탬프 카드는 데이터가 남지 않고, 이탈 위험 고객을 파악하거나 연락할 방법이 없다.

💡 솔루션

QR 코드로 연결되는 디지털 스탬프 카드로 방문 데이터를 자동 수집하고, AI가 이탈 위험 고객을 분류해 재방문 메시지 초안과 SNS 콘텐츠를 자동 생성해준다.

📊 핵심 가치

사장님이 데이터 분석 없이도 "오늘 연락해야 할 고객"을 즉시 파악하고, 버튼 하나로 맞춤형 재방문 메시지를 만들 수 있다.

타겟 사용자

고객 (스탬프 적립자)

  • 카페·베이커리 방문 고객
  • 앱 설치 없이 QR 스캔만으로 적립
  • 전화번호 입력 → 스탬프 즉시 적립

페인포인트

  • 종이 스탬프 카드를 자주 잃어버린다
  • 여러 매장 카드를 지갑에 넣고 다니기 불편하다
  • 어느 매장에 몇 개 모았는지 기억하기 어렵다
  • 앱 설치 없이 간편하게 쓸 수 있으면 좋겠다

사장님 (대시보드 운영자)

  • 독립 카페·베이커리 1~5인 운영
  • 데이터 분석에 익숙하지 않아도 사용 가능
  • 토스플레이스, 자동주문기(키오스크) 등 POS 연동 서비스를 사용하지 않는 매장
  • 현금·수기 운영 비중이 높은 소규모 독립 매장
  • 태블릿 장비 구매 없이 QR 하나로 시작하고 싶은 사장님

페인포인트

  • 단골 고객이 오랫동안 안 나타나도 연락할 방법이 없다
  • 고객 정보를 수기·엑셀·메모앱으로 관리하다 흐지부지된다
  • SNS 포스팅을 자주 올려야 하는데 매번 문구 생각이 어렵다
  • 마케팅 툴은 비싸고 복잡해 배울 시간이 없다
  • 종이 스탬프 카드는 분실·위조 위험이 있고 데이터가 남지 않는다
02 Market Landscape

시장 지형 & 경쟁 분석

소규모 카페·베이커리 스탬프·CRM 시장은 이미 여러 플레이어가 있습니다. 리봇은 기존 서비스들이 커버하지 못하는 "디지털 전환 초기 소규모 독립 매장" 공백을 타겟으로 합니다.

서비스 주요 강점 주요 약점 / 한계 리봇과 관계
토스플레이스
결제 + POS + 포인트 통합 		카드 결제 기반 자동 적립, 별도 앱 불필요, 데이터 자동 집계, 대형 브랜드 레퍼런스 토스 카드 결제 고객만 자동 적립 (현금 고객 제외), POS 교체 비용·학습 허들, AI 재방문 유도 기능 없음 최대 경쟁자
도도포인트
태블릿 기반 디지털 스탬프 		심플한 UX, 스탬프 전문 서비스, 다양한 매장 유형 지원 전용 태블릿 장비 구매 필요, 월 구독료, AI 마케팅 기능 미비, 설치·설정 과정 복잡 간접 경쟁
카카오 멤버십
카카오채널 기반 스탬프 		알림톡 발송 가능, 카카오 생태계 연동, 사용자 친숙도 높음 카카오 생태계 완전 의존, 이탈 분석 없음, AI 메시지 생성 없음, 채널 개설·운영 복잡 간접 경쟁
종이 스탬프 카드
현재 가장 광범위한 방식 		초기 비용 없음, 설치 불필요, 고객에게 익숙 데이터 전혀 없음, 분실·위조 위험, 재방문 유도 불가, 마케팅 연계 불가 대체 대상
리봇(Rebot)
QR 기반 AI CRM 스탬프 		장비 없이 QR 하나로 시작, AI 이탈 감지 + 메시지 자동 생성, 결제 방식 무관, 저비용 도입 알림톡 실발송 미연동(Phase 2), 사장님 인증 미구현(Phase 2) 우리 서비스
리봇의 포지셔닝 — 경쟁사가 놓치는 공백

타겟 매장 조건

  • 토스플레이스 미사용 (현금·수기 결제 비중 높음)
  • 키오스크·자동주문기 미도입
  • 태블릿 장비 구매 의향 없음
  • 월 구독료에 부담 느끼는 초기 단계 매장

리봇만의 차별점

  • QR 출력 + 전화번호 입력만으로 0원 도입
  • 개인화 이탈 감지 (개인 방문 패턴 기반)
  • AI가 메시지·SNS 초안 자동 생성
  • 결제 방식·POS 연동 불필요

경쟁사와 겹치지 않는 이유

  • 토스플레이스 사용 매장은 이미 포인트 적립 가능 → 이탈
  • 도도포인트 사용 매장은 태블릿 있음 → 이탈
  • 종이 카드 사용 매장이 첫 번째 디지털 전환 대상

03 User Flow

서비스 흐름

고객 여정 (QR 스탬프 적립)
1
QR 스캔
매장 내 QR 코드 스캔 → /stamp/[store_code] 접속
2
전화번호 입력
이름 + 전화번호 입력 (SMS 인증 없음). 재방문 시 localStorage에서 자동 불러옴
3
마케팅 동의
첫 방문 시만 표시. 재방문 시 생략
4
즉시 적립
서버에서 즉시 처리 → success / duplicate / error 상태 표시
5
리워드 안내
목표 스탬프 도달 시 리워드 텍스트 표시 (별도 쿠폰 객체 없음)
사장님 여정 (대시보드)
1
대시보드 확인
전체 고객 수, 이탈 현황, 오늘 적립 수 한눈에 파악
2
이탈 위험 고객 확인
AI 이탈 감지 → 주의/위험/장기미방문 단계별 분류
3
AI 메시지 생성
고객 선택 → AI가 맞춤형 재방문 메시지 초안 생성
4
메시지 발송 처리
초안 편집 후 발송 처리 (MVP: 수동, Phase 2: 알림톡 연동)
5
SNS 콘텐츠 생성
이탈 현황 기반 인스타·네이버·카카오 포스팅 초안 자동 생성
04 Feature Scope

기능 범위 — MVP vs Phase 2

초록은 MVP에서 구현하거나 구현 완료, 파란색은 Phase 2 이후 추가할 기능입니다.

고객용 (스탬프 적립 화면)
기능 구분 설명
전화번호 입력 스탬프 적립 ✓ 완료 SMS 인증 없음, 서버 즉시 처리
localStorage 재방문 인식 ✓ 완료 재방문 시 이전 정보 자동 불러오기, 전화번호 마스킹 표시
마케팅 동의 수집 ✓ 완료 첫 방문 시만 표시, 재방문 시 숨김
적립 상태 피드백 ✓ 완료 success / duplicate / error 3가지 상태
리워드 안내 ✓ 완료 목표 스탬프 도달 시 store.reward_desc 텍스트 표시
고객 앱 / 포인트 조회 Phase 2 고객 전용 마이페이지, 스탬프 이력 조회
리워드 쿠폰 관리 Phase 2 만료일, 사용 처리, QR 쿠폰
사장님용 — 대시보드
기능 구분 설명
핵심 지표 요약 카드 MVP 전체 고객, 오늘 적립, 마케팅 동의, 이탈 위험, 최근 30일 재방문, 발송 대기
이탈 현황 4단계 표시 ✓ 완료 정상 / 관심 필요 / 이탈 위험 / 장기 미방문
빠른 작업 버튼 MVP 이탈 고객 보기, 메시지 생성, 콘텐츠 생성, QR 확인
최근 활동 피드 MVP 스탬프 적립, 신규 등록, 초안 생성, 발송 이력
성과 지표 MVP 재방문율, 전환율 — 데이터 부족 시 "측정 중" 표시
사이드바 + 모바일 하단 탭 MVP 데스크톱 사이드바 / 모바일 하단 탭 반응형 레이아웃
사장님용 — 고객 관리
기능 구분 설명
고객 목록 + 이탈 단계 필터 ✓ 완료 전체 / 주의 / 위험 / 이탈 필터링
전화번호 마스킹 ✓ 완료 010****5678 형태로 표시
고객 상세 페이지 ✓ 완료 방문 통계, 스탬프, 방문 이력 타임라인, 메시지 이력
이름·전화번호 검색 MVP 클라이언트 사이드 필터링
마케팅 동의 필터 MVP 미동의 고객은 발송 버튼 비활성화
방문 횟수·최근 방문일 정렬 MVP 컬럼 정렬
고객 직접 등록·편집 Phase 2 사장님이 수동 고객 추가
사장님용 — AI 메시지
기능 구분 설명
AI 재방문 메시지 초안 생성 ✓ 완료 OpenRouter API (Gemini Flash 1.5 8B) 사용, 존댓말·사장님 톤
메시지 편집 + 글자 수 표시 MVP inline 편집, 실시간 카운트
다시 생성 / 복사 / 발송 처리 ✓ 완료 PATCH /api/messages/[id] 로 status='sent' 처리

마케팅 미동의 발송 비활성화 MVP 발송 버튼 disabled + 안내 문구
카카오 알림톡 실제 발송 Phase 2 Solapi API 연동
발송 예약 Phase 2 특정 날짜·시간에 자동 발송
사장님용 — SNS 콘텐츠 / 설정
기능 구분 설명
AI SNS 콘텐츠 초안 생성 ✓ 완료 인스타그램 / 네이버 플레이스 / 카카오채널 채널별 생성
콘텐츠 입력 폼 MVP 홍보 목적, 이벤트 내용, 혜택, 기간, 말투, 강조 포인트
해시태그 자동 제안 MVP 인스타그램 전용
초안 임시 저장 목록 MVP mock 저장 → Phase 2에 DB 연결
SNS 직접 게시 Phase 2 각 플랫폼 API 연동
매장 설정 (이름, 스탬프 목표, 서명) MVP stores 테이블 업데이트
QR 코드 생성·다운로드 MVP 설정 페이지에서 QR 미리보기 + 이미지 저장
매장 로고·브랜드 색상 MVP UI만 구현, 실제 업로드는 Phase 2
사장님 로그인·인증 Phase 2 Supabase Auth, 매장 소유권 검증

09 Algorithm

이탈 감지 알고리즘

lib/churn.ts의 calcChurn(visitedDates: Date[]) 함수가 고객별 이탈 단계를 계산합니다. 방문 데이터를 서버(API Route)에서 수집한 뒤, 각 고객의 방문 이력을 기반으로 개인화된 임계값을 적용합니다.

// lib/churn.ts — 실제 구현 코드 export function calcChurn(visitedDates: Date[]): ChurnResult { if (visitedDates.length === 0) return { churnStage: "safe", daysSinceVisit: 0, avgInterval: null }; const sorted = [...visitedDates].sort((a, b) => b.getTime() - a.getTime()); const daysSinceVisit = Math.floor((Date.now() - sorted[0].getTime()) / 86400000); if (sorted.length === 1) return { churnStage: fixedChurnStage(daysSinceVisit), daysSinceVisit, avgInterval: null }; // 방문 간격 평균 계산 (2회 이상) const intervals = sorted.slice(0, -1).map((d, i) => Math.floor((d.getTime() - sorted[i + 1].getTime()) / 86400000) ); const avgInterval = Math.round(intervals.reduce((a, b) => a + b, 0) / intervals.length); let churnStage: ChurnStage = "safe"; if (daysSinceVisit >= avgInterval * 3) churnStage = "churned"; else if (daysSinceVisit >= avgInterval * 2) churnStage = "danger"; else if (daysSinceVisit >= avgInterval * 1.5) churnStage = "watch"; return { churnStage, daysSinceVisit, avgInterval }; }
💡
개인화 이탈 감지의 장점 — 주 1회 방문하는 고객과 월 1회 방문하는 고객에게 동일한 기준을 적용하면 오탐이 발생합니다. 개인 평균 방문 주기 기반으로 이탈 임계값을 적용하므로, 방문 패턴이 다른 고객을 더 정확하게 분류합니다. 단, 방문이 1회뿐인 신규 고객은 데이터가 부족하므로 고정 임계값(30/60/90일)을 사용합니다.
10 AI

AI 메시지 생성 원칙

메시지 작성 기준
  • 한국어 존댓말, 사장님이 직접 쓴 안부 같은 톤
  • 2~3문장 이내 (카카오 알림톡 길이 고려)
  • 메뉴명이나 결제 금액 직접 언급 금지
  • 과도한 친밀감 또는 감시받는 느낌 배제
  • 매장명 또는 사장님 서명으로 마무리
  • 마케팅 미동의 고객에게는 발송 버튼 비활성화
  • 30일 내 동일 고객 재발송 시 경고 표시

12 Security

보안 고려사항

MVP 단계에서 적용된 보안
  • 전화번호는 DB에 정규화(숫자만 저장), 화면에는 마스킹 표시
  • OpenRouter API Key: 서버(API Route)에서만 사용, 클라이언트 노출 없음
  • Supabase Anon Key: 클라이언트용이나 모든 조회는 서버 경유
  • .env.local은 .gitignore에 포함, 레포에 업로드 금지
  • 마케팅 미동의 고객 메시지 발송 차단 (UI + API)
Phase 2에서 추가할 보안
  • Supabase RLS — 매장별 데이터 격리
  • 사장님 인증 — Supabase Auth
  • store_code 소유권 검증 — 모든 API에 owner 검증 미들웨어
  • Rate Limiting — AI API 호출 남용 방지
  • HTTPS 전용 배포 (Vercel 기본 적용)

MVP 인증 미적용 주의 — 현재 MVP는 사장님 인증 없이 store_code를 알면 대시보드에 접근할 수 있습니다. 내부 테스트 단계에서는 허용되나, 실 서비스 전 Supabase Auth와 RLS 설정이 필수입니다.
