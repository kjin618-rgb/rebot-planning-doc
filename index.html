Strategy Pivot — v2.1 변경 배경

v1에서 출발해 왜 지금의 방향으로 왔는가

v1 서비스 개요 — 이 문서를 처음 보는 분을 위한 배경

리봇 v1은 "카페·베이커리 사장님을 위한 AI 단골 재방문 메시지 엔진"이었다. 핵심 흐름은 단순했다: 사장님이 보유한 고객 데이터(연락처 + 방문이력)를 CSV로 업로드하면, AI가 이탈 위험 고객을 자동으로 감지하고 "사장님이 직접 쓴 것 같은" 개인화 메시지 초안을 생성해준다. 광고성 일괄 문자 대신, 고객 한 명 한 명에게 맞춘 메시지로 재방문을 유도하는 것이 핵심 가치 제안이었다.

v1이 부딪힌 현실: "데이터가 없다"

v1의 치명적 전제는 "사장님이 이미 고객 데이터를 갖고 있다"는 것이었다. 그러나 실제 카페·베이커리 현장을 검토하면, 고객 연락처를 체계적으로 관리하는 소규모 매장은 극히 드물다. 설령 있더라도 이미 다른 CRM 도구를 쓰고 있을 가능성이 높다. v1은 서비스의 핵심 기능(AI 메시지)보다 앞서 해결해야 할 더 근본적인 문제, 데이터 수집 접점 자체가 없다는 허들에 막혔다.

기존 방식의 문제 (v1 → v2)

CSV 업로드: 사장님이 고객 정보를 엑셀로 정리해두는 경우는 극히 드물다. 정리할 만큼 체계적이라면 이미 도도포인트나 다른 도구를 쓰고 있다.


QR 번호 입력(픽업 알림): 2~3분 기다리면 나오는 커피 한 잔을 위해 번호를 입력할 고객은 많지 않다. 고객이 정보를 남길 동기가 없다.

→
새로운 진입점 (v2.1)

디지털 스탬프 카드: 한국 카페 고객은 이미 "10잔 마시면 1잔 무료" 종이 쿠폰에 익숙하다. 이것을 QR 스캔 + 카카오채널로 대체하면, 고객이 자발적으로 정보를 남긴다. 스탬프를 받으려는 동기가 데이터 수집의 엔진이 된다.


핵심 전환: 진입점을 "AI 메시지 서비스"에서 "종이 쿠폰 대체 디지털 스탬프"로 바꾸고, AI 재방문 메시지는 그 위에 올리는 2층 구조.

그러나 새 방향에도 강력한 허들이 있다

디지털 스탬프 카드로의 피벗이 문제를 완전히 해결하지는 않는다. 이 방향에서 반드시 넘어야 할 세 가지 벽이 있다.

허들 1
강력한 직접 경쟁자 이미 존재

도도포인트는 이미 수만 매장에 침투해 있고, 플러터 등 QR 기반 디지털 스탬프 서비스도 존재한다. 사장님 입장에서는 "왜 리봇이어야 하나?"라는 질문에 명확히 답해야 한다. 스탬프 카드 기능만으로는 차별화가 되지 않는다.

허들 2
경쟁 우위 선점의 시간 압박

스탬프 카드는 쉽게 복사된다. 토스플레이스·캐시노트 같은 대형 플레이어가 AI 메시지 기능을 추가하는 순간 리봇의 차별화가 무력화된다. 빠르게 "AI 메시지가 재방문율을 X% 높인다"는 수치 증거를 확보하지 못하면, 진입 기회 자체가 닫힌다.

허들 3
고객 데이터 수집의 실제 어려움

스탬프를 위해 채널을 추가해도 마케팅 동의를 하지 않으면 재방문 메시지 발송 자체가 불법이다. 동의율이 낮으면 DB는 쌓이지만 정작 쓸 수 있는 고객이 적다. 또한 카카오채널에 전적으로 의존하는 구조는 카카오 정책 변경 시 서비스 전체가 흔들리는 리스크를 안는다.

결론 — 무엇을 증명해야 하는가: 리봇은 베타 단계에서 두 가지를 반드시 수치로 증명해야 한다. ① 디지털 스탬프 카드의 마케팅 동의 전환율이 충분히 높은가 (목표 40% 이상), ② AI 메시지가 일반 문자 대비 재방문율을 유의미하게 높이는가 (목표 +20%p). 이 두 수치가 없으면 경쟁사 대비 설득력 있는 포지셔닝이 불가능하다. 반대로 이 수치가 있으면, 도도포인트를 포함한 기존 플레이어에 화이트라벨 제안까지 가능해진다.
01

핵심 문제 정의

카페·베이커리 사장님의 매출은 단골 재방문에 달려 있다. 첫 방문 고객의 주문 단가는 높지만 방문 빈도가 낮아, 실제 매출을 견인하는 것은 반복 방문하는 단골이다. 그런데 대부분의 소규모 매장은 이탈 고객을 인식하는 방법도, 다시 부를 수단도, 데이터를 모을 접점도 없다.

문제 1

이탈 인식 불가

30일 이상 안 온 고객이 누구인지 모름. 느낌으로만 "요즘 뜸하네" 수준.

문제 2

데이터 수집 접점 부재

고객이 자발적으로 연락처를 남길 이유가 없다. 종이 쿠폰은 방문 정보만 있고 누군지는 모른다.

문제 3

데이터 분산

포스기, 도도포인트, 메모장 등 고객 정보가 파편화. 통합 관리 수단 없음.

문제 4

메시지 품질 낮음

광고성 일괄 문자는 고객이 이미 무감각. 개인화 메시지 작성 역량·시간 없음.

리봇의 해답: 종이 쿠폰을 디지털 스탬프로 대체해 고객 DB를 자연스럽게 쌓고, 이탈 위험 고객에게 "사장님이 직접 쓴 것 같은" AI 개인화 메시지로 재방문을 유도한다.
02

타겟 고객

단계별로 타겟을 명확히 구분한다. 초기에는 가장 즉시 작동 가능한 세그먼트에 집중하고, 이후 확장한다.

1
지금~6개월

종이 쿠폰 스탬프를 쓰는 카페·베이커리 사장님

  • 현재 종이 쿠폰(10잔 1잔 무료)을 찍어주고 있는 매장 — 한국 카페의 대다수
  • "디지털로 바꾸면 어떻게 되나요?" 라는 질문에 바로 시연 가능
  • 리봇 QR 스티커 1장으로 즉시 시작, 별도 하드웨어 불필요
  • 연매출 1~3억, 좌석 10~30석, 서울·수도권 중심
2
6~12개월

도도포인트 사용 중 효과에 불만인 사장님

  • 태블릿 설치·유지 비용, 문자 보내도 반응 없다는 불만 보유층
  • 리봇은 하드웨어 없이 QR만으로 대체 가능한 포지션
  • 데이터 이관 지원이 온보딩의 핵심
3
12개월~

B2B: 도도포인트·페이히어·이지포스 등 플랫폼 사업자

  • AI 메시지 엔진을 플랫폼에 탑재하는 화이트라벨 계약
  • 계약 1건 = 수만 사용자 즉시 확보
  • 1·2단계 재방문율 데이터가 협상 레버리지
03

경쟁사 지형

리봇의 경쟁 환경은 데이터를 누가 갖고 있는가, 그리고 사장님의 어떤 습관을 대체하려 하는가를 중심으로 이해해야 한다. 아래 세 카테고리로 구분한다.

직접 경쟁사 — 같은 문제(고객 재방문 유도)를 같은 방식(포인트/스탬프 + 메시지)으로 해결
서비스 핵심 기능 데이터 수집 AI 메시지 하드웨어 비용 위협 수준
도도포인트 태블릿 적립, CRM 문자 발송, 고객 분류 태블릿 직접 입력 (카운터) ❌ 없음 태블릿 필요 유료 (문자 건당) 높음
플러터 QR 기반 디지털 스탬프, 멤버십 관리 QR 스캔 시 자동 ❌ 없음 없음 (QR 스티커) 월 구독 높음 — 진입 방식 동일
리봇 (목표) 디지털 스탬프 + 이탈 감지 + AI 개인화 메시지 QR 스캔 → 카카오채널 추가 ✅ 핵심 차별화 없음 (QR 스티커) 월 구독 —
간접 경쟁사이지만 위험 — 현재는 다른 영역이나 고객 데이터 보유, CRM 기능 확장 가능성 높음
서비스 핵심 기능 데이터 보유 CRM 확장 가능성 위협 이유
토스플레이스 포스 + 결제 + 이탈 쿠폰 자동발송 결제 시 자동 (전화번호) 이미 CRM 진입 중 무료 + 이미 이탈고객 쿠폰 제공. AI 메시지만 없음. 추가 가능성 높음.
캐시노트 매출 분석, AI 비서 '캐시니' 추가 중 카드 결제 자동 수집 AI 기능 추가 중 무료 + 대규모 사용자 기반. AI 메시지 기능 추가 시 직접 경쟁자로 전환.
페이히어 모바일 POS + 소상공인 비즈니스 관리 결제 + 매출 데이터 중간 — 비즈니스 관리 툴 확장 중 소상공인 POS 확장 플레이어. 고객 분석 기능 추가 시 CRM 영역 진입.
이지포스 POS + 재고 관리 + 거래 분석 거래 내역 (메뉴, 금액, 시간) 낮음~중간 — POS 중심이나 데이터는 풍부 이미 거래 데이터 보유. CRM 레이어 추가 시 리봇 진입 영역 잠식 가능.
네이버 스마트플레이스 예약, 리뷰 관리, 단순 알림 예약 시 자동 (이름, 연락처) 중간 — 네이버 생태계 확장 여력 큼 네이버 예약 사용 매장은 이미 고객 DB 존재. 알림 기능 강화 시 진입 여지.
신규 위협 — 현재 직접 경쟁은 아니지만 인프라·생태계 기반으로 빠르게 진입 가능
서비스 현재 포지션 위협 시나리오 대응 방안
카카오비즈니스
(채널 + 알림톡)		 메시지 인프라 제공자 카카오채널에 스탬프 + AI 메시지 기능 직접 탑재. 카카오톡 위에서 모든 것이 해결되면 리봇 필요 없어짐. 리봇이 카카오를 인프라로 쓰되, AI 메시지 품질과 이탈 분석을 핵심 차별화로 유지.
테이블오더 플랫폼
(포잉, 나우버스킹 등)		 QR 기반 테이블 주문 시스템 QR 주문 과정에서 이미 고객 번호를 수집 중. 재방문 메시지 기능 추가 시 리봇과 직접 겹침. 테이블오더를 쓰지 않는 카운터 주문 카페·베이커리에 집중 (대다수의 소규모 매장).
핵심 리스크 재정의: 단일 최대 위협은 토스플레이스다. 이미 무료 결제 데이터 + 이탈 쿠폰 자동발송을 제공한다. AI 메시지를 추가하는 것은 기술적으로 어렵지 않다. 리봇의 생존 포지션은 토스플레이스나 도도포인트가 없는 매장 — 즉 아직 어떤 CRM도 쓰지 않고 종이 쿠폰만 쓰는 카페·베이커리 — 에 먼저 깊이 뿌리내리는 것이다.

리봇이 경쟁사보다 유리한 이유

하드웨어 없음

도도포인트는 태블릿 설치·관리 필요. 리봇은 QR 스티커 1장으로 시작.

앱 설치 없음

고객은 카카오톡만 있으면 됨. 별도 앱 없이 카카오채널로 스탬프 관리.

AI 메시지 품질

쿠폰 자동 발송(경쟁사)이 아닌, 고객별 방문 패턴을 반영한 문장 생성. 이건 경쟁사 핵심 역량 밖.

04

USP 및 차별화 전략

🎫 종이 쿠폰 디지털 전환

사장님은 QR 스티커 1장, 고객은 카카오톡 채널 추가. 앱 설치·태블릿·직원 개입 없이 스탬프 적립 시작. 고객이 자발적으로 데이터를 남기는 구조.

🤖 AI 개인화 메시지 생성

조건 기반 쿠폰 발송(경쟁사)이 아닌, 고객별 방문 패턴을 반영한 문장을 AI가 생성. "N일 미방문 → 쿠폰"이 아닌 "지은 님, 오랜만이에요"가 목표.

📊 이탈 위험 자동 감지

방문 주기 기반으로 이탈 위험도를 자동 계산. 사장님이 따로 확인하지 않아도 "오늘 연락해야 할 고객"을 알려줌.

💰 명확한 ROI

단골 1명 재방문 = 3,600~8,000원. 월 구독료 대비 단골 몇 명이 돌아와야 본전인지 대시보드에 직접 표시.

핵심 USP 두 줄:
"도도포인트는 태블릿이 필요하고, 리봇은 QR 스티커 1장으로 시작한다."
"쿠폰을 뿌리는 게 아니라, 사장님이 직접 쓴 것 같은 편지를 보낸다."

개인화 메시지 설계 원칙

주의 — 개인화 과도 시 불쾌한 협곡(Uncanny Valley) 문제
"지난번 아이스 바닐라 라떼" 같은 메뉴 직접 언급은 일부 고객에게 감시당하는 느낌을 줄 수 있다. AI 메시지는 방문 사실만 환기하고, 메뉴·결제 정보는 직접 언급하지 않는 것이 원칙. "사장님이 손으로 쓴 안부" 톤을 목표로 설계한다.
05

데이터 수집 방안

리봇의 핵심 선결 조건은 고객 연락처 + 방문이력 데이터다. 수집 접점은 고객에게 즉각적인 가치가 있어야 자연스럽게 작동한다. 방안 우선순위는 아래와 같이 변경됐다.

1

디지털 스탬프 카드 → QR 스캔 시 카카오채널 추가 (핵심 진입점)

NEW 핵심

→
카카오채널 추가 + 동의
→
스탬프 1개 적립
→
리봇 DB 자동 적재

종이 쿠폰을 이미 쓰고 있는 매장에서, 그것을 QR로 대체한다. 고객에게는 "카카오로 스탬프 받으세요"라는 한 마디로 설명 완료. 스탬프를 쌓으려는 고객의 자발적 행동이 데이터 수집의 엔진이 된다. 매장 방문이 반복될수록 DB가 자동으로 쌓이는 구조.

법적 요건: 카카오채널 추가와 마케팅 수신 동의는 반드시 분리해야 한다. 채널만 추가하고 마케팅 동의를 안 한 고객에게 재방문 유도 메시지를 보내면 개인정보보호법 위반. 카카오 알림톡 템플릿 심사 승인에 1~2주 소요 — 개발 시작과 동시에 신청 필요.

핵심 전환율 관리 지표

스탬프 카드 수집 방안의 핵심 변수는 채널 추가 중 마케팅 동의 전환율이다. 스탬프를 받기 위해 채널은 추가하지만 마케팅 동의는 안 하는 고객이 많으면, DB는 쌓이지만 재방문 메시지를 보낼 수 있는 고객이 적어진다. 베타 단계에서 반드시 측정해야 할 1순위 지표.

06

MVP 방향성 및 로드맵

리봇은 "더 나은 CRM"이 아니라 "종이 쿠폰을 없애는 도구 + 그 위에 작동하는 AI 메시지 엔진"으로 포지셔닝한다. 풀 CRM 기능 추가는 토스플레이스·도도포인트와의 기능 경쟁을 의미하므로 지양.
1
현재~3개월

Phase 1 — 스탬프 카드 + 데이터 증명

  • QR 스탬프 카드 기능 구현 (카카오채널 연동)
  • CSV 업로드 → 이탈 고객 자동 감지 → AI 메시지 초안 생성 (기존 유지)
  • 베타 사장님 5~10명 무료 운영 (종이 쿠폰 → 디지털 전환 대상 우선)
  • 핵심 측정: AI 메시지 vs 일반 문자의 재방문율 차이
  • 목표: "AI 메시지 재방문율 X% 향상" 수치 확보
  • 개인정보처리방침 + 이용약관 필수 완성
2
3~9개월

Phase 2 — 수집 접점 다양화 + 과금 시작

  • 베이커리 특화 "빵 출고 알림" 기능 추가
  • 카카오 알림톡 API 연동 고도화 (Solapi 또는 NHN Cloud)
  • 재방문 트래킹 기능 (메시지 발송 후 실제 방문 여부 측정)
  • 사장님 대시보드: ROI 자동 계산 (메시지 발송 비용 vs 재방문 매출)
  • 월 구독 과금 시작 (3~5만원/월 예상)
3
9개월+

Phase 3 — B2B 화이트라벨 확장

  • 재방문율 데이터로 도도포인트·페이히어·이지포스에 화이트라벨 제안
  • 플랫폼 API 연동으로 CSV 업로드 의존 탈피
  • 프랜차이즈 본사 단위 계약 검토

08

주요 리스크 및 대응

🔴
토스플레이스 AI 메시지 기능 무료 추가
토스플레이스가 AI 메시지 기능을 무료로 추가하면 리봇의 핵심 차별화가 사라진다. 단, 결제 데이터만 있는 토스는 방문 패턴 분석 깊이에서 스탬프 기반 리봇보다 데이터 질이 낮다. 대응: 스탬프 누적 고객 DB의 양과 질을 빠르게 확보하고, AI 메시지 품질을 수치로 먼저 증명한다.
🔴
마케팅 동의율 저조
스탬프를 위해 채널을 추가해도 마케팅 동의를 안 하면 재방문 유도 메시지 발송 불가. 대응: 마케팅 동의 시 첫 스탬프 2개 증정 등 인센티브 설계로 동의율 극대화. 베타에서 반드시 측정.
🟡
카카오 정책 변경 리스크
카카오채널 정책이나 알림톡 가격이 변경되면 리봇의 핵심 인프라가 흔들린다. 리봇이 카카오에 강하게 의존하는 구조는 장기적으로 취약점이다. 대응: 카카오를 인프라로 쓰되, 고객 데이터는 리봇 자체 DB에 저장. 중장기적으로 네이버 톡톡 등 대안 채널 이중화 검토.
🟡
플러터 등 직접 경쟁사의 AI 기능 추가
디지털 스탬프 카드 분야 경쟁사가 AI 메시지 기능을 추가하면 진입 장벽이 낮아진다. 대응: AI 메시지 품질과 재방문율 데이터를 가장 빠르게 쌓아 "리봇 = 재방문율 X% 증가"라는 수치 기반 신뢰를 확보한다.
🟡
개인화 과도 → 불쾌한 협곡
메뉴·결제 정보 직접 언급 시 일부 고객이 거부감. 대응: AI 메시지 원칙을 "방문 환기 + 따뜻한 인사" 톤으로 제한. 메뉴명 직접 언급 지양.
🟢
카카오 알림톡 템플릿 반려
광고성 문구로 판단되면 심사 반려. 대응: 스탬프 적립 알림과 마케팅 메시지를 명확히 분리하고, 적립 알림은 비광고성 톤으로 설계.
